# CLAUDE.md

이 문서는 Claude Code가 이 레포에서 작업할 때 항상 참고해야 하는 프로젝트 운영 규칙입니다.

## 워크플로우 (Agentic Engineering 패턴)

이 프로젝트는 아래 4단계 순서를 반드시 지켜 진행합니다.

1. **Explore** — Context 인식 단계. `prd.md`(제품 요구사항), `spec.md`(기술 스펙)를 먼저 작성/참조하여 프로젝트 맥락을 파악한다.
2. **Plan** — 구현 계획 수립. `plan.md`에 거시적(macro) 계획과 Phase별 세부 계획을 모두 작성한다.
3. **Action** — 실제 구현.
4. **Commit** — 품질적인 검토 후 커밋한다.

## 커밋 룰

- **Plan 단계**: 계획 수립이 끝나면 반드시 커밋을 남긴다. 이 커밋에는 `plan.md`(거시적 + Phase별 계획)가 포함되어야 한다.
- **Action 단계**: 구현이 끝나면 반드시 커밋을 남긴다.

### 커밋 메시지 형식

`[type] 커밋 내용` 형식을 따른다. 커밋 메시지 내용은 한글로 작성한다.

| type | 용도 |
|---|---|
| `[feat]` | 기능 추가 |
| `[refact]` | 리팩토링 |
| `[fix]` | 버그 수정 |
| `[test]` | 테스트 추가/수정 |
| `[chore]` | 마이너한 텍스트/위치 수정 등 |
| `[docs]` | `prd.md`/`spec.md`/`plan.md` 등 문서 작성/수정 |

## 참고 문서

- `prd.md` — 제품 요구사항 (무엇을 만들 것인가)
- `spec.md` — 기술 스펙 (아키텍처, 데이터 모델 등)
- `plan.md` — 구현 계획 (거시적 + Phase별)
