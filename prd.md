# PRD — 반도체 시료 생산주문관리 시스템

## 1. 배경 및 문제 정의

가상의 반도체 기업 **S-Semi**는 연구소, 팹리스 업체, 대학 연구실 등을 대상으로 다양한 반도체 시료(Sample)를 제작해 납품한다. 지금까지는 주문이 들어오면 담당자가 설비를 통해 시료를 만들고 검수 후 출고하는 흐름을 엑셀과 메모장으로 관리해왔다.

그런데 최근 주문 건수가 크게 늘면서 다음과 같은 문제가 반복적으로 발생하고 있다.

- 특정 주문이 처리되었는지 여부를 확인하기 어렵다.
- 공정을 예약했지만 완료 시점을 추적할 방법이 없다.
- 재고가 이미 충분함에도 불필요한 추가 생산이 진행되는 경우가 있다.

수기 관리 방식으로는 재고와 공정 현황을 한눈에 파악할 수 없어 실수가 잦다. 이를 해결하기 위해 S-Semi는 **반도체 시료 생산주문관리 시스템**을 자체 개발하기로 했다.

## 2. 이해관계자(역할)

| 역할 | 책임 |
|---|---|
| 고객 | 필요한 시료를 이메일 등으로 요청 |
| 주문 담당자 | 요청 내용에 맞춰 주문서를 작성·관리, 승인/거절 결과를 고객에게 전달 |
| 생산 담당자 | 신규 시료를 시스템에 등록, 접수된 주문의 재고를 확인해 승인 또는 거절, 생산 진행 |

세 역할의 업무 흐름: 고객이 주문 담당자에게 시료를 요청 → 주문 담당자가 주문서를 작성해 생산 담당자에게 전달 → 생산 담당자가 재고를 확인해 승인/거절을 결정하고, 그 결과가 다시 주문 담당자에게 돌아온다.

## 3. 시스템 개요

- 실행 형태: **콘솔(터미널) 기반** 애플리케이션. 화면 레이아웃 자체는 자유롭게 구성 가능.
- 생산 라인: 공장 내 시료 하나를 만들어내는 단일 설비 흐름으로 가정한다. 한 라인은 한 번에 시료 1건씩만 생산하며, 실제로 주문이 들어온 시료에 대해서만 생산이 발생한다.
- 조작 방식: 담당자가 메뉴 번호와 값을 직접 입력해 시료 등록, 주문 처리 등을 수행한다.

## 4. 주문 상태 정의

모든 주문은 아래 다섯 가지 상태 중 하나를 갖는다. `REJECTED`는 정상 흐름에서 벗어난 상태이므로 이후 모니터링 대상에서는 제외한다.

| 상태 | 의미 |
|---|---|
| `RESERVED` | 주문이 막 접수되어 승인 대기 중 |
| `REJECTED` | 생산 담당자가 주문을 거절함 |
| `PRODUCING` | 승인은 되었으나 재고가 부족해 부족분을 생산 중 |
| `CONFIRMED` | 승인 완료, 재고 확보되어 출고 대기 중 |
| `RELEASE` | 출고가 완료됨 |

### 상태 전이도

```
RESERVED --(승인, 재고충분)--> CONFIRMED --(출고)--> RELEASE
RESERVED --(승인, 재고부족)--> PRODUCING --(생산완료)--> CONFIRMED --(출고)--> RELEASE
RESERVED --(거절)--> REJECTED
```

## 5. 기능 요구사항

시스템 진입 시 전체 시료에 대한 요약 정보를 보여주는 메인 메뉴에서 아래 6개 기능으로 진입할 수 있어야 한다.

### 5.1 시료 관리
시료(Sample)는 이 시스템의 최소 관리 단위다. 시스템에 등록된 시료만 주문할 수 있다.
- **등록**: 새 시료를 추가한다. 필요한 속성 값은 시료 ID, 이름, 평균 생산시간, 수율이다.
  - 수율 = 정상 생산된 시료 수 / 총 생산 시료 수 (예: 100개 중 90개 정상이면 수율 0.9)
- **조회**: 등록된 시료 전체 목록과 현재 재고 수량을 함께 보여준다.
- **검색**: 이름 등 속성으로 특정 시료를 찾는다.

### 5.2 시료 주문
고객의 요청을 받아 주문 담당자가 주문(예약)을 생성한다.
- 입력 값: 시료 ID, 고객명, 주문 수량
- 생성 직후 주문 상태는 `RESERVED`가 된다.

### 5.3 주문 승인 / 거절
`RESERVED` 상태 주문 목록을 확인하고 건별로 승인 또는 거절을 결정한다.
- **승인**: 재고 상황에 따라 자동으로 분기 처리한다.
  - 재고 충분 → 즉시 `CONFIRMED`로 전환
  - 재고 부족 → 생산 라인에 자동 등록, `PRODUCING`으로 전환
- **거절**: 즉시 `REJECTED`로 전환

### 5.4 모니터링
- **주문량 확인**: 상태별(`RESERVED`/`CONFIRMED`/`PRODUCING`/`RELEASE`) 건수를 집계한다. `REJECTED`는 유효 주문이 아니므로 집계에서 제외.
- **재고량 확인**: 시료별 현재 재고 수량과, 주문 대비 재고 수준을 아래 3단계로 표시한다.
  - 여유: 주문 대비 재고 충분
  - 부족: 주문 대비 재고 수량 모자람
  - 고갈: 재고 수량 0

### 5.5 생산 라인 조회
`PRODUCING` 상태 주문의 부족분을 실제로 생산하는 로직이다.
- 실 생산량 = `ceil(부족분 / 수율)`
- 총 생산 시간 = 평균 생산시간 × 실 생산량
- 생산이 끝나면 해당 주문 상태를 `PRODUCING → CONFIRMED`로 전환한다.
- 현재 생산 중인 시료 정보와, 생산 대기열(생산 큐)에 쌓인 주문 목록을 함께 보여준다. 대기열 처리 순서는 **선입선출(FIFO)** 이다.
- 화면에 노출할 세부 정보(주문 정보, 누적 생산량 등)의 수준은 자유롭게 결정한다.

### 5.6 출고 처리
`CONFIRMED` 상태 주문 중 하나를 선택해 출고를 실행한다.
- 처리 완료 시 상태를 `CONFIRMED → RELEASE`로 전환한다.

## 6. Out of Scope (이번 과제 범위 밖)

- 다중 생산 라인은 고려하지 않는다.
- 고객 역할은 구현 대상이 아니다.
